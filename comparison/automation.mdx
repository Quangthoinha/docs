# Workflow Automation

Tự động hóa quy trình là bước đệm đầu tiên để đưa AI vào thực tế. Dưới đây là phân tích chi tiết bộ 3 công cụ phổ biến nhất hiện nay.

## Bảng so sánh nhanh (Quick Specs)

| Tính năng | Activepieces | n8n | Make.com |
| :--- | :--- | :--- | :--- |
| **Language** | TypeScript | TypeScript | Closed Source |
| **Self-hosted** | ✅ Có (Docker) | ✅ Có (Docker/npm) | ❌ Không (SaaS) |
| **Pricing** | Open Source (Free core) | Fair-code (Free self-host) | Tính theo Ops (Đắt) |
| **Mức độ khó** | ⭐⭐⭐ (Trung bình) | ⭐⭐⭐⭐ (Khá khó) | ⭐⭐ (Dễ) |

## Tổng quan công cụ

<CardGroup cols={3}>
  <Card title="Activepieces" icon="bolt">
    **Best for Open Source**
    Trình duyệt automation mã nguồn mở phát triển nhanh nhất. Nhẹ, hỗ trợ MCP và cộng đồng active.

  <Card title="n8n" icon="workflow">

  <Card title="Make" icon="cloud">

    Giao diện "bong bóng" trực quan nhất thế giới. Ổn định, nhiều tích hợp sẵn, nhưng giá chát khi scale.
  </Card>
</CardGroup>

## Phân tích sâu (Deep Dive)

<AccordionGroup>
  <Accordion title="Ưu điểm và Nhược điểm của Activepieces">
    **Ưu điểm:**
    *   Mã nguồn mở (MIT License), bạn thực sự sở hữu dữ liệu.
    *   UX đơn giản hơn n8n, sạch sẽ.
    *   Hỗ trợ AI Agent native khá tốt.

    **Nhược điểm:**
    *   Sinh sau đẻ muộn, số lượng Pieces (integrations) ít hơn Make.
    *   Chưa có tính năng xử lý lỗi (Error handling) mạnh như n8n.
  </Accordion>

  <Accordion title="Ưu điểm và Nhược điểm của n8n">
    **Ưu điểm:**
    *   Linh hoạt tuyệt đối: Code node cho phép làm mọi thứ.
    *   Workflow phức tạp: Hỗ trợ Loop, If-Else, Switch, Merge dữ liệu cực mạnh.
    *   LangChain Integration: Có node LangChain tích hợp sẵn để build AI.

    **Nhược điểm:**
    *   Learning Curve cao: Không dành cho người sợ code.
    *   Resource Heavy: Bản self-host ngốn RAM nếu không tối ưu.
  </Accordion>

  <Accordion title="Ưu điểm và Nhược điểm của Make">
    **Ưu điểm:**
    *   Cực kỳ trực quan, dễ hiểu luồng đi của dữ liệu.
    *   Error handling (Ignore, Rollback) rất trực quan.
    *   Hệ sinh thái App khổng lồ.

    **Nhược điểm:**
    *   Giá rất đắt khi chạy hàng nghìn operations mỗi ngày.
    *   Không thể Self-host (Dữ liệu phải đi qua server của Make).
  </Accordion>
</AccordionGroup>

<Warning>
  **Phán quyết (Verdict)**
  
  *   Chọn **Make** nếu bạn có ngân sách, cần sự ổn định ngay lập tức và quy trình không quá phức tạp.
  *   Chọn **n8n** nếu bạn là team kỹ thuật, cần xử lý dữ liệu phức tạp, bảo mật dữ liệu khách hàng (Self-host) và tích hợp sâu AI.
  *   Chọn **Activepieces** nếu bạn thích Open Source, muốn nhúng automation vào sản phẩm của mình (Embed).
</Warning>
